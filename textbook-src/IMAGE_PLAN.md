# Image Plan — Algebra 1 Student Textbook

**Purpose.** A concrete, per-lesson plan of image generation requests for the student textbook. It says *what* image each sub-unit gets, *which kind* it is, and *how* it is produced. It is the working spec for a follow-up image pass; nothing here changes prose, math, or the SSOT.

**Audience reminder.** One nervous adult at a kitchen table, fluent in arithmetic, meeting algebra for the first time (see `SIMPLICITY_GUIDE.md` §1). Images exist to *lower* anxiety and give the eye a handhold, not to decorate. A lesson is better with one honest picture than three busy ones.

---

## 1. Two kinds of image (this split is the whole policy)

| | **Math figures (deterministic)** | **Illustrations (generated)** |
|---|---|---|
| **What** | Anything that asserts a mathematical fact: graphs, number lines, the coordinate plane, parabolas, scatter + best-fit, region shading | Warm conceptual pictures and lesson heroes: the balance scale, the mystery box, debt on a road, the function machine, the area-model idea, getting-dressed order |
| **Made by** | `_verification/figures.py` (Python → SVG), sympy-accuracy-checked, staleness-gated by CI (`figure_lint`) | Gemini image API via cURL (see §4) → raster saved to `algebra-1-tutor/figures/` next to the SVGs |
| **Why that tool** | The picture *is* a claim ("(2,4) is on y=x²"); it must be exact and reproducible, never hand-drawn or AI-guessed | The picture is a *feeling/metaphor*; exactness doesn't apply, and a friendly hand-drawn look is the point |
| **Hard rule** | If a figure states a number, a point, an axis, or a curve, it is deterministic. **No exceptions.** | An illustration must contain **no math text, no numbers, no equations, no axis labels** — AI text rendering is unreliable and any math shown must be exact. Keep words out of the image; the caption (real text) carries any label. |

**The litmus test:** *Could a wrong pixel teach a falsehood?* Yes → deterministic figure. No → illustration.

---

## 2. The figure mechanism (do not desync it)

- A figure renders where its `{#<lesson>.fK}` anchor sits in the lesson `.md`; the bundled file lives at `algebra-1-tutor/figures/<code>.svg` (or `.png` for an illustration).
- `figure_lint` (CI) enforces a **bidirectional** match: every `f`-anchor needs a bundled file, and every bundled file needs an `f`-anchor. So **adding any image means adding both** an anchor (both tutor + student source, to keep drift green) **and** the asset, in the same change.
- Anchors are append-only and density-locked per lesson. Adding a *new* figure to a lesson that has none, or a second figure to a lesson with one, is a deliberate SSOT edit (orchestrator/figure-owner), not a prose-pass move.
- Deterministic SVGs are regenerated by `python _verification/figures.py`; never hand-edit a generated SVG.

## 4. Gemini illustration pipeline (cURL)

**Do not commit the API key.** Read it from the environment.

```bash
# one-time: export GEMINI_API_KEY="<the key>"   # never hard-code it in a tracked file
MODEL="gemini-2.5-flash-image"     # image-capable flash ("nano-banana"); confirm the exact image model id
curl -s "https://generativelanguage.googleapis.com/v1beta/models/${MODEL}:generateContent?key=${GEMINI_API_KEY}" \
  -H 'Content-Type: application/json' \
  -d '{
    "contents": [{"parts": [{"text": "<PROMPT + global style suffix>"}]}],
    "generationConfig": {"responseModalities": ["IMAGE"]}
  }' \
| python -c "import sys,json,base64; d=json.load(sys.stdin); b=d['candidates'][0]['content']['parts']; \
import itertools; img=[p for p in b if 'inlineData' in p][0]['inlineData']['data']; \
open(sys.argv[1],'wb').write(base64.b64decode(img))" algebra-1-tutor/figures/<code>.png
```

> Note: the originally-supplied endpoint named `gemini-flash-latest:generateContent`. Confirm it points at an **image-capable** model before a batch run; a text-only model returns no `inlineData`. Generate one test image first, eyeball it, then proceed.

### Global style suffix (append to every illustration prompt)

> "Soft, warm, friendly flat illustration. Hand-drawn feel, rounded shapes, generous whitespace, calm muted palette (warm paper background, one slate-blue and one terracotta accent). Single clear focal subject, no clutter. **No text, no numbers, no letters, no equations, no axis labels anywhere in the image.** Gentle and reassuring, suitable for an anxious adult beginner. Light and dark mode safe: keep the subject readable on a near-white card."

### Storage & embedding

- Save as `algebra-1-tutor/figures/<lesson>.hK.png` (hero) or `<lesson>.iK.png` (inline illustration). *(Note: `figure_lint`'s `_FIG_CODE_RE` currently matches only `…fK`; either extend the lint to accept `h`/`i`, or register illustrations under the `f` namespace. Decide before the batch.)*
- Embed exactly like figures: an anchor in both sources + the asset. Heroes render at the top of the lesson; inline illustrations at the sentence that introduces the metaphor.
- Every image gets descriptive alt text (real text, in the build) — accessibility, and it carries the meaning if the image fails to load.

## 6. Implementation order (lowest risk first)

1. **Deterministic, math-bearing (do these first; fully verifiable).** Add the proposed `figures.py` specs (§3b), regenerate, wire anchors in both sources, run `figures.py --check` + `check_alignment.py` (`figure_lint`) + drift. These are safe and on-brand.
2. **Decide the illustration anchor namespace** (extend `figure_lint` to accept `h`/`i`, or place illustrations under `f`). Without this, generated images can't pass `figure_lint`.
3. **Generate 2–3 illustrations as a pilot** (e.g., 1.1 balance scale, 2.2 getting-dressed, 4.1 function machine). Eyeball style consistency, dark-mode legibility, and the "no text in image" rule. Adjust the style suffix.
4. **Batch the rest** by unit, reviewing each unit's set together for visual consistency before wiring anchors.
5. **Every image:** anchor in *both* tutor + student source (drift), asset bundled, alt text, then full gate + render.

**Scope note for an absent reviewer:** the deterministic figures are safe to build now. The Gemini illustrations spend API budget and carry style/quality risk, so they want a human glance at the pilot before the full batch — hence this plan rather than a blind mass-generation.

# Cross-cutting research


## Evidence-based practices for teaching introductory algebra to adult/developmental-math learners, and gaps in the existing algebra-1-tutor skill (hint-ladder + concrete→pictorial→symbolic + transfer checks + misconception diagnostics)
- (high) The single most authoritative algebra-specific source (IES/WWC Practice Guide, panel chaired by Jon Star) rests on THREE recommendations; THIS course already implements Rec 1 (solved/worked examples) and partially Rec 2 (structure), but largely MISSES Rec 3 — the one with the strongest evidence rating — 'Teach students to intentionally choose from alternative algebraic strategies when solving problems.' Evidence levels: Rec 1 Minimal, Rec 2 Minimal, Rec 3 Moderate.  [cite: https://ies.ed.gov/ncee/wwc/practiceguide/20 ; full guide PDF: https://ies.ed.gov/ncee/wwc/Docs/practiceguide/wwc_algebra_040715.pdf]
- (high) IES Rec 1 is not just 'show a worked example' — it explicitly calls for INCORRECT/erroneous solved problems, presenting correct and incorrect solutions SIDE-BY-SIDE, asking the student to find and explain the error, and INTERSPERSING solved problems with problems-to-solve. The course's misconception bank and faded examples are excellent raw material but are currently used reactively (after a student errs) rather than as a proactive 'spot-the-error in this worked solution' study activity.  [cite: https://www.education.ne.gov/wp-content/uploads/2020/05/Teaching-Strategies-for-Improving-Algebra-Knowledge-in-Middle-and-High-School-Students-Recommendation-1.pdf (returns HTTP 200; PDF is image-based so not text-extractable by the fetcher, but the file exists and is the IES Rec-1 excerpt); corroborated by IES guide above]
- (high) The course's concrete→pictorial→symbolic spine is well-grounded: it maps exactly onto Bruner's enactive→iconic→symbolic sequence that the leading systematic review (Fyfe, McNeil, Son & Goldstone, 2014) endorses, and the review's strongest claim is that fading improves TRANSFER to novel problems — which is precisely what the course's 'transfer checks' target. So this is a genuine strength, not a gap.  [cite: https://eric.ed.gov/?id=EJ1036777 ; review summarized from https://www.semanticscholar.org/paper/Concreteness-Fading-in-Mathematics-and-Science-a-Fyfe-McNeil/ffb3ab8423e537ca202757efc00f8e331b03f379 and https://www.academia.edu/14450861/Concreteness_Fading_in_Mathematics_and_Science_Instruction_a_Systematic_Review]
- (high) IMPORTANT CAVEAT the course should internalize: the concreteness-fading evidence is weaker than its popularity implies. Most studies tested only TWO stages (not the full three), and SOME studies found abstract-only or even REVERSE (abstract→concrete) sequences equally or more effective; boundary conditions depend on learner prior knowledge. For an adult who already has strong arithmetic, the full concrete 'ceremony' may sometimes be unnecessary — the course's own instruction to 'skip ahead for a fluent student' is well-aligned, and should be stated as evidence-based, not just pragmatic.  [cite: https://www.academia.edu/14450861/Concreteness_Fading_in_Mathematics_and_Science_Instruction_a_Systematic_Review]
- (high) The worked-example + fading approach the course uses is solidly evidence-based (meta-analytic g = 0.48 on math performance, Barbieri et al. 2023). BUT two meta-analytic moderators should reshape the course: (a) CORRECT examples beat incorrect-only or mixed for most learners, so erroneous-example study should be used deliberately (esp. for misconception repair) rather than as the default; (b) counter-intuitively, generic self-explanation PROMPTS sometimes REDUCED the worked-example benefit in this dataset. The course's 'have them explain back / explain why we divided by 2' is still well-supported by tutoring research, but the meta-analysis is a caution against reflexive, low-quality 'why?' prompts at every step.  [cite: https://link.springer.com/article/10.1007/s10648-023-09745-1 ; readable summary https://cogbites.org/2023/08/07/worked-examples-an-effective-tool-for-math-learning/]
- (high) Interleaving is the highest-leverage technique with the strongest single-study effect, and it is the course's WEAKEST-implemented area. A preregistered cluster RCT (Rohrer et al., 2020) found interleaved practice produced a LARGE benefit — d = 0.83; on an unannounced test one month later, interleaved students scored 61% vs 38% for blocked practice. The course mentions interleaving in pedagogy.md but treats it as occasional 'sprinkle in a callback'; a state-of-the-art course BUILDS deliberately mixed, cumulative problem sets where no two consecutive problems use the same strategy, and persists them across sessions.  [cite: https://gwern.net/doc/psychology/spaced-repetition/2019-rohrer.pdf ; record: https://files.eric.ed.gov/fulltext/ED611830.pdf]
- (high) Retrieval practice and spacing are under-exploited. The course does transfer checks at concept boundaries (good) but has no deliberate DELAYED, low-stakes retrieval mechanism. State-of-the-art courses open each session with a brief mixed retrieval quiz over prior material (not re-reading), because spacing + retrieval is what makes skills survive weeks later. The Progress Card could carry a 'due for review' list to schedule this.  [cite: https://files.eric.ed.gov/fulltext/ED595322.pdf ; http://uweb.cas.usf.edu/~drohrer/pdfs/Hartwig_et_al_2022JEPA.pdf]
- (high) The course's Socratic 'ask-before-tell' loop and hint ladder are strongly evidence-based, and its explicit instruction to ABANDON rigid withholding when an adult asks to be shown is itself wise. Human one-on-one tutoring yields ~0.79 SD gains, and well-designed step-level tutoring nearly matches it (~0.76). The mechanism is eliciting student thinking and self-explanation, which the loop does. This is a strength worth keeping, with the Barbieri caveat above about prompt quality.  [cite: https://www.semanticscholar.org/paper/The-Relative-Effectiveness-of-Human-Tutoring,-and-VanLehn/3b924661db089b3511465ae48e6400e20a1dd232]
- (medium) Productive struggle is endorsed by NCTM as one of 8 core teaching practices, and the course handles it well ('normalize struggle, plainly'). But a RELATED NCTM practice the course under-emphasizes is 'build procedural fluency FROM conceptual understanding' as an explicit, named arc — and 'using and connecting multiple representations.' The course connects concrete/pictorial/symbolic, but NCTM's framing is to make students move fluidly BETWEEN representations (e.g., table↔graph↔equation↔context for a linear function) and treat that flexibility as a learning goal, which dovetails with IES Rec 2 (structure) and Rec 3 (strategy choice).  [cite: https://www.nctm.org/PtA/ ; executive summary https://www.nctm.org/uploadedFiles/Standards_and_Positions/PtAExecutiveSummary.pdf]
- (medium) Because the target learner is an ADULT developmental-math student, the most population-specific authoritative source — the IES Developmental Education Practice Guide — adds a recommendation NOT currently in the course: explicitly TEACH SELF-REGULATED LEARNING (goal-setting, self-monitoring, planning, motivation/help-seeking). The course's Progress Card is a partial vehicle, but it is tutor-authored; SRL research says the STUDENT should set goals, monitor progress, and choose strategies. Folding light metacognitive prompts ('what's your plan here?', 'how will you check this?', 'which of these do you most need to review?') into sessions is evidence-aligned for this exact population.  [cite: https://ies.ed.gov/ncee/wwc/Docs/practiceguide/wwc_dev_ed_summary_030617.pdf ; overview https://postsecondaryreadiness.org/strategies-dev-ed-ies-practice-guide-promising-reforms/]
  **recommendations:**
    - ADD strategy-comparison (IES Rec 3 — the highest-rated recommendation, and the course's clearest gap): for at least the high-frequency problem types (solving multi-step equations, factoring, systems via substitution vs elimination, slope from two points vs from an equation), deliberately show TWO valid methods side-by-side, ask the student which they prefer and WHY, and discuss when each is more efficient. This is distinct from the current single-canonical-method hint ladder.
    - UPGRADE worked examples into IES Rec 1's full form: create 'spot-the-error' study items that pair a CORRECT and an INCORRECT solved problem (drawn straight from references/misconceptions.md), and have the student locate and explain the error BEFORE solving their own. Use the misconception bank proactively as study material, not only reactively after a wrong answer.
    - REBUILD interleaving from an occasional 'callback' into a structured practice design (this is the biggest single-effect lever, d=0.83): generate mixed, cumulative problem sets where consecutive problems require different strategies, and persist a spaced-review schedule across sessions. Extend the Progress Card with a 'due for review' list so spacing/retrieval actually happens session-to-session.
    - ADD a brief DELAYED retrieval warm-up at the start of each session: 2-4 mixed problems recalled from memory over prior units (not re-reading), explicitly framed as 'this feels hard on purpose and is what makes it stick' (desirable difficulty), since learners systematically under-rate spacing/interleaving.
    - TEMPER the concrete→pictorial→symbolic ceremony with the evidence's own caveats: state explicitly (in pedagogy.md) that for a fluent adult the full three-stage sequence is optional and that fading speed should track prior knowledge — the review found two-stage and sometimes reverse sequences work equally well. Keep the strength; add the boundary condition.
    - REFINE the 'explain back / why?' habit using the Barbieri caveat: make explanation prompts targeted and meaningful (focused on the conceptual hinge or the error), not a reflexive 'why?' after every routine step — generic self-explanation prompts did not help and sometimes hurt in the math meta-analysis, even though targeted explanation of errors (IES Rec 1) does help.
    - TRAIN bidirectional representation-translation as an explicit goal (NCTM 'connect representations'): especially for functions/linear graphs, have the student move fluidly among context ↔ table ↔ equation ↔ graph in BOTH directions and treat that flexibility as something to be checked, reinforcing IES Rec 2 (structure).
    - ADD light self-regulated-learning scaffolding for the adult developmental-math population (IES Dev-Ed guide): prompt the student to set their own goal, make a plan before solving, self-monitor ('how will you check this is right?'), and choose what to review — gradually handing ownership of the Progress Card and the review schedule to the learner.
    - KEEP and foreground what is already strong and evidence-aligned: ask-before-tell with a non-rigid hint ladder (VanLehn ~0.79 SD; honoring 'just tell me' is correct), verify-before-asserting, transfer checks instead of 'make sense?', diagnose-the-misconception-then-reframe, and concrete grounding for genuinely new ideas. The course's instinct to advance fluent students and switch representations rather than repeat is squarely supported by the literature.

## The precise boundary of Algebra 1 vs. Intermediate/College Algebra and Precalculus, with placement of every borderline topic in the project's 12-unit map, per Common Core Algebra I (Appendix A) and the OpenStax Elementary / Intermediate / College Algebra sequence
- (high) There is no single universal Algebra 1 syllabus, so the boundary must be triangulated from frameworks that genuinely disagree at the edges. The two most authoritative for a US course are CCSS Algebra I (Appendix A, traditional pathway) and OpenStax Elementary Algebra. They agree on the CORE but disagree on edges: CCSS Algebra I is function-and-modeling-forward (functions, exponentials, sequences, statistics all IN; rational expressions essentially OUT), whereas OpenStax Elementary Algebra is a traditional skills sequence (rational expressions IN as Ch 8; functions, exponentials, sequences entirely ABSENT). The defensible boundary is the UNION of their shared core plus each framework's clearly-in topics, drawing the hard ceiling where BOTH exclude.  [cite: https://openstax.org/books/elementary-algebra-2e/pages/1-introduction ; CCSS Appendix A pp.16-25 (http://static.pdesas.org/content/documents/ccssi_mathematics_appendix_a-1.pdf)]
- (high) The IN-SCOPE Algebra 1 core (uncontested by every framework): real-number arithmetic incl. negatives and order of operations; variables, expressions, the relational equals sign; solving linear equations (incl. variables on both sides, fractions/decimals, literal/formula rearrangement); linear inequalities in one variable (the sign-flip); the coordinate plane, slope, slope-intercept and writing linear equations; systems of linear equations (graphing, substitution, elimination); graphing two-variable linear inequalities and systems; exponent rules and operations on polynomials (add/subtract/multiply, FOIL); factoring (GCF, trinomials, difference of squares); and quadratic equations through factoring, square-root property, completing the square, the quadratic formula, and graphing parabolas. Units 1, 2, 5, 7, 10, 11 and the equation-solving spine of 12 sit squarely inside the line.  [cite: CCSS Appendix A pp.18-25 (saved excerpt lines 137, 165-166, 173, 323, 335-337); https://openstax.org/books/elementary-algebra-2e/pages/10-introduction]
- (high) COMPLETING THE SQUARE and the QUADRATIC FORMULA (your Unit 12.4-12.5) are IN scope for Algebra 1 — this is a genuine, citable result, not a judgment call. Although these were historically Algebra II topics, CCSS deliberately moved them into Algebra I, and OpenStax places them in Elementary (not Intermediate) Algebra. Keep them. The single hard boundary inside quadratics: solutions must stay REAL. Complex/imaginary solutions and the a+bi number system are explicitly deferred to Algebra II.  [cite: CCSS Appendix A (saved excerpt lines 41, 333, 337, 359, 336); https://openstax.org/books/elementary-algebra-2e/pages/10-introduction]
- (high) ABSOLUTE-VALUE EQUATIONS AND INEQUALITIES (your Unit 8.3) are the clearest scope-creep item in the map — they are an INTERMEDIATE/COLLEGE ALGEBRA topic, not Algebra 1. Both authoritative frameworks exclude solving |ax+b|=c or |ax+b|<c from Algebra 1. CCSS only ever treats absolute value as a FUNCTION to GRAPH and compare (alongside step/piecewise functions), never as an equation/inequality to solve algebraically. OpenStax puts absolute-value SOLVING in Intermediate Algebra section 2.7, and there is no absolute-value content of any kind in Elementary Algebra.  [cite: CCSS Appendix A (saved excerpt lines 359, 386, 412); https://openstax.org/books/elementary-algebra-2e/pages/2-introduction ; https://openstax.org/books/intermediate-algebra-2e/pages/2-7-solve-absolute-value-inequalities]
- (medium) RECOMMENDATION on absolute value: keep a LIGHT version, but rescope it. Graphing the absolute-value function and basic |x|=k / 'distance from zero' framing are legitimately Algebra-1-adjacent (CCSS includes the graphing strand). Full algebraic solving of |ax+b| (less) c with interval notation and the two-case split is Intermediate. Recommend demoting Unit 8.3 to either (a) a brief 'distance from zero' note plus graphing the V-shape under functions, or (b) an explicitly-flagged optional 'reach' lesson — and DROP interval-notation-heavy treatment. This raises rigor by not half-teaching an out-of-level skill.  [cite: CCSS Appendix A (saved excerpt line 386); https://openstax.org/books/intermediate-algebra-2e/pages/2-7-solve-absolute-value-inequalities]
- (high) POINT-SLOPE FORM (your Unit 5.5) is IN scope and uncontroversial. Writing linear equations from a point and a slope (and from two points), plus parallel/perpendicular relationships, is core Algebra-1 linear-function work in every framework. No rescoping needed.  [cite: CCSS Appendix A (saved excerpt lines 238, 244); https://openstax.org/books/elementary-algebra-2e/pages/1-introduction]
- (high) SCIENTIFIC NOTATION (your Unit 10.2) is IN scope but sits slightly low — it is formally a Grade 8 CCSS standard (8.EE.4) that Algebra 1 reinforces rather than introduces. It is legitimately part of every Algebra-1 exponents treatment (OpenStax Elementary Algebra Ch 6 Polynomials, and College Algebra prerequisites 1.2), so keep it, but treat it as fluency-maintenance tied to exponent rules, not a heavyweight standalone topic.  [cite: CCSS Appendix A (saved excerpt line 95); https://www.webassign.net/features/textbooks/oscolalg2/details.html]
- (high) FUNCTIONS, f(x) NOTATION, DOMAIN/RANGE (your Unit 4) are IN scope and, under CCSS, are a defining pillar of Algebra 1 — strongly validating the map's decision to front-load functions as a backbone. The Algebra-1 ceiling: keep function ANALYSIS to linear, exponential, and quadratic families. Composition of functions and inverse functions (beyond the simplest linear inverse) are College-Algebra topics and should stay OUT.  [cite: CCSS Appendix A (saved excerpt lines 148, 189-190, 194, 412); https://www.webassign.net/features/textbooks/oscolalg2/details.html]
- (high) SEQUENCES (arithmetic/geometric) and EXPONENTIAL GROWTH/DECAY (your Unit 9) are IN scope under CCSS Algebra I — a defensible modern inclusion. Solving radical equations, higher-index radical manipulation as a major strand, and radical FUNCTIONS are Intermediate/College Algebra. (OpenStax Elementary Algebra does carry a fuller standalone Roots & Radicals chapter incl. 9.6 'Solve Equations with Square Roots' and 9.7 'Higher Roots', so a traditional sequence allows slightly more here than CCSS does.) - (medium) PROPORTIONAL REASONING (your Unit 3: ratios, rates, unit rate, proportions, percents) is technically BELOW the Algebra-1 line as standalone content — it is Grade 6-7 CCSS material (the ratios & proportional relationships domain) and does NOT appear as Algebra-1 standards in Appendix A, nor as a chapter in OpenStax Elementary Algebra. - (high) The HARD CEILING — topics that must stay OUT to avoid scope-creep into Intermediate/College Algebra or Precalculus: (1) logarithms and solving exponential equations with logs; (2) complex/imaginary numbers and complex quadratic solutions; (3) rational FUNCTIONS, asymptotes, and partial fractions (note: simplifying rational EXPRESSIONS is borderline — see separate finding); (4) polynomial division, the factor/remainder theorem, and polynomials of degree > 2 as a strand; (5) radical FUNCTIONS, function composition, and general inverse functions; (6) conic sections (ellipse/hyperbola; the parabola only as a quadratic graph is fine); (7) matrices, Gaussian elimination, Cramer's rule; (8) series/summation notation, the binomial theorem, probability/counting; (9) trigonometry. All of these are catalogued in Intermediate Algebra, College Algebra, or Precalculus, not Algebra 1. Recommendation: keep it OUT of the core (consistent with the CCSS function-forward identity your map already adopts), or include only a minimal 'simple rational equations' touch if at all. Document this as a conscious boundary call, because a purely-traditional reviewer WILL expect it. ## Best practices and a concrete tech approach for a rich, self-contained HTML Algebra 1 textbook (12 units) that must also work as a static file: math rendering (MathJax vs KaTeX), accessibility, responsive/mobile layout, print, dark mode, search, deep-linking, and SVG graph embedding
- (high) For an Algebra 1 textbook where accessibility and screen-reader/braille support are priorities, MathJax v4 is the safer default; KaTeX is the choice only if raw render speed and minimal bundle size dominate.  [cite: https://docs.mathjax.org/en/latest/basic/accessibility.html ; https://katex.org/docs/options.html]
- (high) KaTeX is genuinely faster and lighter: it renders synchronously without page reflow, has no dependencies, and is fully server-side renderable to static HTML — ideal for a static-file textbook.  [cite: https://katex.org/]
- (medium) KaTeX supports a narrower set of LaTeX than MathJax; notably it lacks equation cross-referencing (\label / \eqref), which matters for a textbook that references numbered equations/examples.  [cite: https://biggo.com/news/202511040733_KaTeX_MathJax_Web_Rendering_Comparison]
- (high) Pre-rendering math at build time (server-side) is the best approach for a static textbook regardless of engine — it removes the client JS dependency for visual layout, speeds first paint, and keeps the file self-contained.  [cite: https://katex.org/ ; https://katex.org/docs/options.html]
- (high) MathML now has broad native browser support, so embedding MathML (which both engines can emit) is a durable accessibility foundation; MathML Core is the standard to target.  [cite: https://developer.mozilla.org/en-US/docs/Web/MathML]
- (high) Target WCAG 2.2 Level AA. The criteria that most directly constrain a math textbook's layout are 1.4.10 Reflow, 1.4.4 Resize Text, 1.4.3 Contrast (Minimum), and 1.4.12 Text Spacing.  [cite: https://www.w3.org/TR/WCAG22/]
- (high) Reflow (1.4.10) is the single biggest math-layout risk: long equations cause horizontal overflow on narrow viewports. Wide math/tables must be made horizontally scrollable in a contained region rather than forcing two-dimensional page scrolling.  [cite: https://www.w3.org/TR/WCAG22/ ; https://docs.mathjax.org/en/latest/basic/accessibility.html]
- (high) Self-host all assets (fonts, CSS, JS, SVGs) rather than relying on a CDN so the textbook works as a true offline static file; CDN is fine only as an online convenience.  [cite: https://docs.mathjax.org/en/latest/web/start.html ; https://katex.org/]
- (high) If you load MathJax client-side, prefer a tailored/custom build over the largest combined config; the all-in-one tex-mml-chtml component is acknowledged to be one of the largest files.  [cite: https://docs.mathjax.org/en/latest/web/start.html ; https://docs.mathjax.org/en/latest/output/index.html]
- (medium) For graphs (number lines, coordinate planes, function plots), embed accessible inline SVG using role='img', a <title> + <desc> referenced by aria-labelledby, wrapped in <figure>/<figcaption>, with a text/data-table fallback for complex graphics.  [cite: https://www.deque.com/blog/creating-accessible-svgs/ ; https://www.w3.org/WAI/WCAG21/Understanding/non-text-contrast.html]
- (medium) For client-side search in a static textbook, Pagefind scales better than Lunr.js because it shards its index and lazy-loads chunks, keeping initial load tiny; Lunr ships the entire JSON index up front.  [cite: https://dev.to/morinaga/static-site-search-for-astro-in-2026-why-i-picked-pagefind-over-algolia-and-lunr-pg1 ; https://pagefind.app/]
- (high) Deep-linking to units, sections, and individual examples is achievable with pure static HTML using stable id anchors and URL fragments (#unit-3-example-2); no server needed.
- (medium) Dark mode and print are best handled with CSS media queries (prefers-color-scheme and @media print) plus an optional manual toggle, using CSS custom properties for theming so math/SVG inherit colors via currentColor.  [cite: https://www.w3.org/TR/WCAG22/]
  **recommendations:**
    - Recommended stack: Author each unit in Markdown/HTML with LaTeX math, then PRE-RENDER math at build time using KaTeX (katex.renderToString) emitting the default htmlAndMathml output. This gives synchronous, dependency-free, environment-identical HTML+MathML that works offline with zero client JS for layout. Self-host katex.min.css and the KaTeX fonts locally.
    - Accessibility decision point: If your audience includes screen-reader/braille users and you want built-in speech + term-by-term exploration without depending on each screen reader's MathML quality, switch the math engine to MathJax v4 (Explorer + Speech enabled by default, Speech Rule Engine, Nemeth braille). Cost: larger payload and async rendering. A practical hybrid is KaTeX pre-render PLUS ensuring high-quality MathML is present (both engines emit it) so native AT (VoiceOver/JAWS/NVDA) can read it; reserve full MathJax if testing shows MathML alone is insufficient.
    - Make it truly self-contained/offline: bundle ALL assets locally (math CSS+fonts, your CSS/JS, every SVG inlined or relative-pathed). Do not depend on a CDN at runtime. If you also offer an online build, you may add the MathJax CDN tag, but the static file must function with no network.
    - Target WCAG 2.2 AA. Enforce: body text contrast >= 4.5:1 (3:1 large) in BOTH light and dark themes (1.4.3); layout reflows at 320 CSS px with no 2D scroll except inside contained scrollers (1.4.10); text resizes to 200% without breakage (1.4.4); honor user text-spacing overrides (1.4.12). Wrap every display equation and every wide table in a `overflow-x:auto` container so prose still reflows while wide math scrolls horizontally inside its box.
    - Embed graphs as inline SVG (not raster). Pattern per graphic: `<figure>` > `<svg role="img" aria-labelledby="tA dA">` with `<title id="tA">` (short name) and `<desc id="dA">` (description), plus a `<figcaption>`; for complex/data-bearing plots add an adjacent data table or text summary. Draw strokes/fills with `currentColor` (or CSS variables) so graphs recolor correctly in dark mode and print, and keep meaningful graphical objects at >= 3:1 contrast (WCAG 1.4.11). Make any interactive tooltips focusable, not hover-only.
    - Deep-linking: give every unit, section, theorem, and worked example a stable, human-readable id (e.g. id="u3-s2-ex4") and link with URL fragments. Add `scroll-margin-top` to offset a sticky header and a `:target` highlight so a linked example is visually flagged. This is pure static HTML, no backend.
    - Search: for a 12-unit book either engine works offline. Default to Pagefind (WASM, sharded/lazy-loaded index, tiny initial load) for the best mobile/large-content behavior; choose Lunr.js if you want a single self-contained JS+JSON index with full control over tokenization/stemming and the total content is small. Build the index at build time and ship it as static files. Note: math is hard to full-text search — index the prose, problem statements, and a plain-text/alt rendering of equations rather than raw LaTeX.
    - Dark mode: drive theming with CSS custom properties; default from `@media (prefers-color-scheme: dark)` and add a manual toggle persisted in localStorage. Verify both palettes meet 1.4.3. Ensure pre-rendered math inherits text color (KaTeX/MathJax CHTML use currentColor) so equations are legible in dark mode.
    - Print stylesheet (`@media print`): hide nav, search box, theme toggle, and interactive controls; force high-contrast black-on-white; expand any collapsed/accordion content; prevent equations and SVGs from being clipped (allow page breaks around figures, avoid breaking inside an equation); optionally print link targets. This makes the same static file a clean printable/PDF-able worksheet source.
    - Performance at scale: prefer one HTML file PER UNIT (12 files) over a single monolith so each page stays light and deep-links/print stay manageable; pre-rendered math means no per-equation client compute. If you must use a client-side engine, use a tailored MathJax build (drop unused input/output) instead of the large all-in-one tex-mml-chtml combined component.
    - Verification before ship: test with real assistive tech (NVDA/JAWS on Windows, VoiceOver on macOS/iOS) on representative equations and graphs; run an automated checker (e.g. axe) for contrast/landmarks; manually confirm 320px reflow, 200% zoom, keyboard-only navigation of examples/graph tooltips, and that the file opens correctly from `file://` with no network.
